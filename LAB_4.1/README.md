Лабораторная работа №4
**Тема: «Синхронизация процессов»**

Описание программы
Система межпроцессной коммуникации через общий файл с использованием кольцевой очереди FIFO.

Архитектура системы:
Процесс Receiver (Приемник)
Основные функции:
Создание и управление бинарным файлом для сообщений
Запуск и контроль процессов Sender
Чтение сообщений из общей очереди
Ожидание готовности отправителей
Управление состоянием системы

Логика работы:
Инициализация файла с фиксированными слотами
Мониторинг заполненности очереди
Блокировка при пустом буфере
Обработка команд с консоли

Процесс Sender (Отправитель)
Основные функции:
Открытие общего файла для записи
Отправка уведомлений о готовности
Запись сообщений в очередь
Ожидание освобождения места в буфере

Логика работы:
Генерация сообщений длиной до 20 символов
Блокировка при заполненном буфере
Работа по командам с консоли
Корректное завершение

Механизм синхронизации:
Кольцевая очередь FIFO:
Фиксированное количество слотов в файле
Циклическое использование буфера
Атомарные операции чтения/записи
Отслеживание индексов head/tail
Примитивы синхронизации:
Флаги готовности процессов
Бинарные семафоры для доступа к буферу
Счетчики занятых слотов
Механизмы ожидания событий

Особенности реализации:
Для упрощенной версии (1 Receiver + 1 Sender):
Единовременная передача одного сообщения
Прямая связь без конкурентного доступа
Упрощенная логика синхронизации
Базовые операции блокировки
